# Capstone Project Evaluation Rubric

This rubric defines the criteria used to evaluate the quality, clarity, and completeness of the project's definition, design, and delivery.

## 1. Define (10%)

| Aspect                                          | Max Score | Scoring Guide                                                                     |
| ----------------------------------------------- | --------- | --------------------------------------------------------------------------------- |
| **Business Context, Stakeholders, and Value**   | **/ 10**  | **10:** All details thoroughly covered.                                           |
| - Explanation of the business context.          |           | **9:** Most details covered.                                                      |
| - Understanding and engagement of stakeholders. |           | **8:** Some aspects covered, some missing.                                        |
| - User needs are well defined.                  |           | **7:** Problem not placed in context that well.                                   |
|                                                 |           | **5 or below:** Shows a lack of attention to the business context of the problem. |

## 2. Design (45%)

| Aspect                                                                                                      | Max Score | Scoring Guide                                                                                                                                 |
| ----------------------------------------------------------------------------------------------------------- | --------- | --------------------------------------------------------------------------------------------------------------------------------------------- |
| **Git / README**                                                                                            | **/ 10**  | **10:** README thorough and Git shows excellent use of versioning including branches and releases.                                            |
| - README gives a detailed description of the project and how to install and run the program.                |           | **8/9:** README thorough and Git well used.                                                                                                   |
| - Git goes beyond basic versioning, making use of releases and feature branches.                            |           | **7:** Git does not use branches, only one line of development.                                                                               |
|                                                                                                             |           | **6 or below:** Git only used as a remote drive, not for version control.                                                                     |
| **Documentation**                                                                                           | **/ 15**  | **10:** Thoroughly documented report and code.                                                                                                |
| - Using the appropriate level of details to document the problem, stakeholders, and solution.               |           | **8/9:** Report and/or code missing key details.                                                                                              |
| - Organisation and structure of documentation and code.                                                     |           | **7:** Room for improvement in documentation.                                                                                                 |
|                                                                                                             |           | **6 or below:** Documentation inadequate.                                                                                                     |
| **Project Planning**                                                                                        | **/ 10**  | **9/10:** Clear articulation of the planning process and steps needed to bring the project into production (e.g. list of tasks, Gantt chart). |
| - The project planning, effort allocation, and next steps.                                                  |           | **8:** Shows understanding of design steps, some items missing.                                                                               |
| - Understanding of the effort used to perform the design work and remaining effort to complete the project. |           | **7 or below:** Minimal project planning documented.                                                                                          |
| **Testing**                                                                                                 | **/ 10**  | **10:** Complete testing with the report documenting these.                                                                                   |
| - Your application has unit tests for all the functionalities.                                              |           | **8:** Unit tests for functionalities.                                                                                                        |
| - API functionality tests included.                                                                         |           |                                                                                                                                               |

## 3. Delivery (45%)

| Aspect                                                                                           | Max Score | Scoring Guide                                                                                                                                                                                                   |
| ------------------------------------------------------------------------------------------------ | --------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Quality of Server**                                                                            | **/ 10**  | **9/10:** Effective implementation of MVC model with appropriate use of classes.                                                                                                                                |
| - The code reflects the MVC model (route/controller etc) and classes are used where appropriate. |           | **8:** Code is done adequately and commented.                                                                                                                                                                   |
| - The code is tidy and commented properly.                                                       |           | **7:** Web server is used correctly but room for improvement.                                                                                                                                                   |
| **User Interface**                                                                               | **/ 10**  | **9/10:** User interface is responsive and of high quality. Code is neat, well documented, the UI shows signs of research and the application is well constructed. A HI-FI (or close to) prototype is provided. |
| - The application is well constructed and attractive.                                            |           | **8:** The user interface reflects some basic research and effort.                                                                                                                                              |
|                                                                                                  |           | **7:** The user interface is basic.                                                                                                                                                                             |
| **Functionalities**                                                                              | **/ 10**  | **10:** All functions work as intended.                                                                                                                                                                         |
| - The functions exist and work as intended.                                                      |           | **8/9:** Most functions work as intended, some room for improvement.                                                                                                                                            |
| **Database**                                                                                     | **/ 5**   | **5:** Able to perform create/delete operations as well as simple queries.                                                                                                                                      |
| - Makes use of a database with CRUD operations.                                                  |           | **3:** Makes minimal use of a database.                                                                                                                                                                         |
| **Live Demo, Poise and Audience Engagement**                                                     | **/ 10**  | **10:** Excellent engaging presentation with clarity and well answered questions from audience.                                                                                                                 |
| - Ability to deliver a clear, concise and engaging presentation.                                 |           | **9:** Excellent presentation with room for improvement in clarity and/or handling of questions.                                                                                                                |
| - Ability to listen effectively and address questions.                                           |           | **8:** Very good presentation, could be more engaging.                                                                                                                                                          |
| - Overall poise, confidence and rapport with the audience.                                       |           | **7:** Adequate presentation, lacking clarity.                                                                                                                                                                  |
| - Keeping the time.                                                                              |           | **6 or below:** Poor delivery.                                                                                                                                                                                  |
|                                                                                                  |           | _(Deduct 1 for each 2 minutes beyond the allocated time range)_                                                                                                                                                 |
